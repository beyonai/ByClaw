# 架构设计

本文档介绍 ByClaw 的整体架构设计。

## 架构概览

ByClaw 采用微服务架构，由多个独立模块组成，通过 API 和消息队列进行通信。

## 核心模块

| 模块 | 职责 | 技术栈 |
|------|------|--------|
| [byclaw-fe](../byclaw-fe/) | Web 前端 | React 18, Umi Max 4, TypeScript 5 |
| [byclaw-be](../byclaw-be/) | 核心后端服务 | Spring Boot 3.4, Java 21 |
| [byclaw-exe](../byclaw-exe/) | 扩展插件和技能 | Python, TypeScript |
| [byclaw-data](../byclaw-data/) | 数据云服务（Agent 编排） | Python 3.12, uv |
| [byclaw-qa](../byclaw-qa/) | QA 管理和 Agent 服务 | Python 3.12, uv |

## 部署架构

支持多种部署模式：

- **单体模式** - 适合快速体验和小规模部署
- **拆分模式** - 适合生产环境，各服务独立扩展
- **Kubernetes** - 适合大规模云原生部署

详见 [部署文档](../deployment/)。

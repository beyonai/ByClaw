# 数据流图

本文档描述 ByClaw 核心业务的数据流转过程。

## 用户认证流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  用户   │────▶│  前端   │────▶│  后端   │────▶│ 数据库  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
                                      │                │
                                      │◀───────────────┘
                                      │
                                      ▼
                               ┌─────────┐
                               │  Redis  │
                               │ 会话缓存 │
                               └─────────┘
```

**流程说明：**
1. 用户输入用户名密码
2. 后端验证凭据
3. 生成 JWT Token 和 Refresh Token
4. 会话信息存入 Redis
5. 返回 Token 给前端
6. 前端存储 Token，后续请求携带

## AI 对话流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  用户   │────▶│  前端   │────▶│  后端   │────▶│  AI模型 │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
                    │                 │                │
                    │                 │                │
                    │◀────────────────┼◀───────────────┘
                    │    WebSocket    │   流式响应
                    │    实时推送      │
                    ▼
               ┌─────────┐
               │ 用户界面 │
               │ 逐字显示 │
               └─────────┘
```

**流程说明：**
1. 用户发送消息
2. 后端组装上下文（历史消息 + 系统提示）
3. 调用 AI 模型（支持流式响应）
4. 后端接收流式输出，实时转发给前端
5. 前端 WebSocket 接收，逐字显示
6. 对话结束后保存到数据库

## 知识库文档处理流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  用户   │────▶│  前端   │────▶│  后端   │────▶│  MinIO  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
                                      │                │
                                      │                │
                                      ▼                │
                               ┌─────────┐             │
                               │ OpenGauss│◀────────────┘
                               │ 元数据   │
                               └────┬────┘
                                    │
                                    │ 异步通知
                                    ▼
                            ┌───────────────┐
                            │  byclaw-data  │
                            │   处理服务    │
                            └───────┬───────┘
                                    │
                    ┌───────────────┼───────────────┐
                    │               │               │
                    ▼               ▼               ▼
              ┌─────────┐    ┌─────────┐    ┌─────────┐
              │ 文本提取 │    │ 分块处理 │    │ 向量化  │
              │         │───▶│         │───▶│         │
              └─────────┘    └─────────┘    └────┬────┘
                                                  │
                                                  ▼
                                           ┌─────────┐
                                           │ OpenGauss│
                                           │ 向量存储 │
                                           └─────────┘
```

**流程说明：**
1. 用户上传文档（PDF/DOCX/TXT/MD）
2. 后端接收文件，存入 MinIO
3. 文档元数据存入 OpenGauss
4. 发送异步处理任务
5. byclaw-data 服务处理：
   - 从 MinIO 下载文档
   - 提取纯文本内容
   - 文本分块（Chunking）
   - 调用向量化模型生成 Embeddings
   - 存入 OpenGauss 向量表
6. 处理完成后更新文档状态

## RAG 检索流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  用户   │────▶│  后端   │────▶│ 向量化  │────▶│ OpenGauss│
│ 提问    │     │         │     │ 服务    │     │ 向量检索 │
└─────────┘     └─────────┘     └─────────┘     └────┬────┘
                                                      │
                    ┌─────────────────────────────────┘
                    │ 返回 Top-K 相关文档
                    ▼
┌─────────┐     ┌─────────┐     ┌─────────┐
│  用户   │◀────│  后端   │◀────│  AI模型 │
│ 收到回答 │     │ RAG组装 │     │ 生成回答 │
└─────────┘     └─────────┘     └─────────┘
```

**流程说明：**
1. 用户提问
2. 将问题向量化
3. 在向量数据库中检索相似文档块
4. 组装 RAG Prompt（上下文 + 检索到的文档 + 问题）
5. 调用 AI 模型生成回答
6. 返回给用户

## 文件上传流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│  前端   │────▶│  后端   │────▶│  MinIO  │
│ 选择文件│     │ 预签名  │     │ 直接上传 │
└─────────┘     └─────────┘     └─────────┘
     │                               │
     │      直接上传                 │
     │◀──────────────────────────────┘
     │
     ▼
通知上传完成
```

**流程说明：**
1. 前端请求预签名上传 URL
2. 后端生成 MinIO 预签名 URL
3. 前端直接使用预签名 URL 上传文件到 MinIO
4. 上传完成后通知后端
5. 后端保存文件元数据

## 数字员工执行流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  用户   │────▶│  后端   │────▶│ Agent   │────▶│ 工具调用 │
│ 触发任务 │     │ 接收请求 │     │ 编排引擎 │     │ byclaw-exe│
└─────────┘     └─────────┘     └─────────┘     └────┬────┘
                    ▲                                │
                    │         返回结果                │
                    └────────────────────────────────┘
```

**流程说明：**
1. 用户触发数字员工任务
2. 后端加载 Agent 配置（提示词、工具、工作流）
3. Agent 编排引擎规划执行步骤
4. 按需调用外部工具（byclaw-exe 扩展）
5. 汇总结果返回给用户

# 数字员工使用指南

数字员工（AI Agent）是可配置的 AI 助手，能够执行特定任务和工作流。

## 什么是数字员工？

数字员工是：
- 🎯 **角色化** - 拥有特定身份和专业领域
- 🛠️ **工具化** - 可调用各种工具和 API
- 🤖 **自动化** - 可自动执行预设工作流
- 📚 **知识化** - 可绑定专属知识库

## 使用场景

| 场景 | 说明 |
|------|------|
| 客服助手 | 自动回答常见问题 |
| 数据分析师 | 分析数据并生成报告 |
| 内容创作 | 辅助写作和内容生成 |
| 代码助手 | 编程辅助和代码审查 |

## 创建数字员工

### 1. 基础配置

1. 点击 **"数字员工"** 导航
2. 点击 **"新建员工"**
3. 填写基础信息：
   - **名称** - 员工名称
   - **头像** - 选择或上传头像
   - **角色描述** - 定义身份和能力
   - **欢迎语** - 首次对话的问候语

### 2. 角色定义

在 **"系统提示词"** 中详细描述：

```
你是一位专业的客户服务专员，擅长：
1. 解答产品使用问题
2. 处理常见售后咨询
3. 收集客户反馈

回复风格：
- 友好、耐心
- 简洁、清晰
- 使用中文回复
```

### 3. 关联知识库

选择员工可以访问的知识库：
- 产品手册知识库
- 常见问题知识库
- 公司政策知识库

### 4. 配置工具

启用员工可以使用的工具：
- **查询订单** - 查询客户订单信息
- **创建工单** - 创建售后服务工单
- **发送邮件** - 发送邮件通知

## 工作流配置

### 什么是工作流？

工作流定义了数字员工处理任务的步骤：

```
用户请求 → 意图识别 → 信息收集 → 工具调用 → 结果输出
```

### 创建工作流

1. 进入员工详情页
2. 切换到 **"工作流"** 标签
3. 点击 **"编辑工作流"**
4. 添加流程节点：
   - **开始节点** - 接收用户输入
   - **条件节点** - 判断分支
   - **工具节点** - 调用工具
   - **结束节点** - 返回结果

### 示例工作流

**客服工单流程：**

```
[开始] → [识别问题类型] → [收集必要信息] → [创建工单] → [通知用户] → [结束]
                ↓
           [查询知识库]
```

## 使用数字员工

### 在对话中使用

1. 进入 **"对话"** 页面
2. 点击 **"选择员工"**
3. 选择要对话的数字员工
4. 开始对话

### 分享数字员工

1. 进入员工详情页
2. 点击 **"分享"**
3. 选择分享方式：
   - 链接分享
   - 嵌入代码
   - 二维码

## 管理数字员工

### 监控使用情况

- 对话次数统计
- 用户满意度评分
- 工具调用成功率
- 常见问题分析

### 优化改进

根据数据优化员工：
1. 更新知识库内容
2. 调整系统提示词
3. 添加新的工具
4. 优化工作流逻辑

### API 调用

通过 API 调用数字员工：

```bash
curl -X POST http://api.example.com/v1/agent/chat \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "agentId": "agent-123",
    "message": "你好"
  }'
```
